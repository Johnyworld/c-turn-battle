## 1. Feature Overview

### 1.1 Feature Name
`hex-turn-battle` — 헥스 격자 턴제 3진영 전투 게임

### 1.2 Goal
Node.js 서버가 정적 파일을 서빙하고, 웹 브라우저에서 Canvas API로 동작하는 2차 세계대전 배경의 턴제 전략 게임.
- **연합군(factionId 0)**: 플레이어가 직접 클릭으로 조작
- **추축군(factionId 1), 소련군(factionId 2)**: AI가 자동으로 턴 진행
- **Fog of War**: 연합군 유닛의 시야 범위 밖은 어둡게 가려 적 위치 미표시

### 1.3 Background / Motivation
- `npm start` 한 줄로 로컬 서버를 띄워 브라우저에서 접속하는 방식
- 파일 분리로 게임 로직(JS 모듈), 스타일(CSS), 마크업(HTML)을 명확히 분리
- 향후 서버 기능(세이브, 점수판 API 등) 확장에 용이한 구조
- 2차 세계대전의 실제 유닛 유형(탱크, 야포, 보병 등)으로 역사적 분위기 구현

## 3. Scope

### 3.1 In Scope
- 헥스 격자 맵 렌더링 및 좌표 시스템
- 3개 진영 색상 구분 및 기지 배치
- 5종 유닛의 이동/공격 로직
- 턴제 진행 시스템 (진영 순환)
- 유닛 간 전투 (데미지 계산, HP 감소, 사망 처리)
- 기지 공격 및 패배 조건
- 전투 로그 표시
- **추축군·소련군 규칙 기반 AI 자동 조종** (`ai.js`)
- **Fog of War 시야 계산 및 렌더링** (`fog.js`)

### 3.2 Out of Scope
- AI 고급 전략 (A* 경로 탐색, 진영 간 협력 등)
- 플레이어가 AI 진영 선택
- 네트워크 멀티플레이
- 유닛 생산/경제 시스템
- 세이브/로드 기능
- 사운드/BGM

## 4. Technical Stack

| 영역 | 기술 |
|------|------|
| 서버 | Node.js (내장 `http` 모듈, 별도 프레임워크 없음) |
| 렌더링 | HTML5 Canvas API |
| 언어 | Vanilla JavaScript ES Modules (프론트), CommonJS (서버) |
| 프로젝트 구조 | `server.js` + `public/` 디렉터리 분리 |
| 스타일 | `public/style.css` 외부 파일 |
| 실행 | `npm start` → `http://localhost:3000` |
| 배포 | 로컬 서버 / 향후 클라우드(Railway, Render 등) 가능 |

    └── js/

        ├── render.js      # Canvas 렌더링 (Fog 오버레이 포함)

## 5. Unit Specifications

## 7. Turn System

```
턴 순서: 연합군 → 추축군 → 소련군 → (반복)

연합군 턴 (플레이어):
  1. 유닛 클릭으로 선택
  2. 이동 가능 헥스 하이라이트 → 클릭으로 이동
  3. 공격 가능 타깃 하이라이트 → 클릭으로 공격
  4. 모든 유닛 행동 완료 또는 "턴 종료" 버튼 클릭
  5. 다음 진영으로 전환

추축군·소련군 턴 (AI):
  1. 해당 진영의 살아있는 유닛 순회
  2. 각 유닛별 행동 자동 결정 및 실행 (300ms 딜레이)
     - 의무병: 인접 아군 HP < 50% 이면 회복
     - 야포: 사거리 내 적 있으면 공격, 없으면 적 방향 이동
     - 일반: 공격 가능 타깃 있으면 공격, 없으면 이동 → 재공격 시도
  3. 모든 유닛 처리 후 자동으로 다음 턴 전환
  4. AI 턴 중 플레이어 입력 차단

승리 조건:
  - 마지막 1개 진영만 기지가 살아있으면 승리
```

## 7.1 Fog of War

```
가시 영역 = 연합군 유닛 각각의 시야(반경 3헥스) 합집합
           + 연합군 기지 위치 (항상 가시)

비가시 영역:
  - 헥스 타일: 어두운 오버레이 표시
  - 적 유닛·기지: 렌더링 생략 (위치 미공개)
  - 아군 유닛: 항상 표시

Fog 토글: 헤더 버튼으로 ON/OFF 전환 가능
```

## 9. Risks

| 위험 | 대응 |
|------|------|
| 헥스 좌표 변환 복잡도 | Offset 좌표 방식 사용, `hex.js`에 단위 테스트 수준 검증 |
| ES 모듈 CORS 오류 | 서버 없이 파일 직접 열면 CORS 발생 → `npm start` 필수 안내 |
| AI 행동 품질 | 규칙 기반 단순 AI — 공격 우선, 기지 방향 이동으로 합리적 도전감 제공 |
| AI 턴 타이밍 | async/await + setTimeout으로 순차 처리, 렌더링과 충돌 없음 |
| Fog 계산 성능 | 턴 시작 시 1회 계산 후 캐싱, 유닛 이동 시 재계산 |
| Canvas 성능 | 이벤트 발생 시에만 `render()` 호출 (requestAnimationFrame 불필요) |
